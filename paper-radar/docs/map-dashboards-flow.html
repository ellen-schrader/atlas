Redesign proposal · Flow 2

Maps become documents you keep.

Today the map is one page and one throwaway configuration. This flow turns it into a library of saved, shareable map dashboards — create by exploring live data, not by filling in a form. The pinned Overview stays; your own maps live beside it.

Configuration as chips

The saved lens shows as the same chips used across Atlas — the state is on the page, not buried.

Saves the lens, not a snapshot

Reopening always shows the current corpus. The +12 since last visit delta turns recall into a reason to return.

Share with the lab

Personal by default; sharing puts it in every member's library under your avatar. Map switcher (breadcrumb ▾) hops between maps in one click.

Exploring ≠ editing

Hover and click-through don't dirty the map; only rail changes do, surfacing Update / Save as new / Reset.

Why this is the right-sized change

01Lives up to the name

An atlas is a collection of maps. Today we ship one. This makes the product match its own promise — and its metaphor do real work.

02Nearly free to build

One maps table with RLS and CRUD. Every pixel of rendering already exists in Map.tsx and the /overview endpoint.

03Turns effort into memory

The interesting views a PI builds today vanish on navigation. Saving makes them recallable, shareable, and a reason to come back.
